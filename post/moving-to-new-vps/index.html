虚拟主机大搬家

December 9, 2018

几天前收到Bandwagon的邮件，我的VPS因Open VZ技术问题停止服务。

得，赶紧买新的。年付27美刀的 2018 Black Friday Special V3 CN2 套餐，就你了。到手首先装个Debian 9（我真是受够了陈旧的 Centos 6 x86 了，趁着买新的机器赶紧换掉）。

安装V2Ray

鉴于Shadowsocks已经被盯上，笔者选择转投技术比较新的V2Ray：

apt-get update
apt-get install unzip
apt-get install curl
bash <(curl -s -L https://233blog.com/v2ray.sh)

然后选择安装，输入1，回车；

接下选择传输协议，笔者选择8，伪装为微信通话（如果被运营商Qos的话请切回TCP）；

选择端口，老规矩挪用Minecraft的19132端口；

开启广告拦截，并且不配置Shadowsocks。这一部分算是大功告成。

安装Transmission

说实话，其实这里才是重头戏，笔者在安装transmission这一步卡了整整一个下午（时间都去哪儿了？笑）。好，废话少说：

apt-get transmission-daemon -y

这时候你要是急着打开浏览器登录ip:9091，你只能看到这么几行字儿：

403: Forbidden

Unauthorized IP Address.

Either disable the IP address whitelist or add your address to it.

If you’re editing settings.json, see the ‘rpc-whitelist’ and ‘rpc-whitelist-enabled’ entries.

If you’re still using ACLs, use a whitelist instead. See the transmission-daemon manpage for details.

子曰：欲速，则不达。

/etc/init.d/transmission-daemon stop

打开 /var/lib/transmission-daemon/info/settings.json ，修改好 rpc-username 和 rpc-password 两项，并把 rpc-whitelist 和 rpc-whitelist-enabled 两项直接删掉。这时候再打开ip:9091，可以看见Transmission已经可以正常工作了。但是因为默认的下载文件夹藏得比较深，不方便往后的一些操作，所以笔者选择把下载地址修改为 /home/Downloads。最后，别忘了授权并启动：

mkdir -p /home/Downloads
chmod -R 755 /home/Downloads
chown -R debian-transmission /home/Downloads
chgrp -R debian-transmission /home/Downloads
/etc/init.d/transmission-daemon start

安装apache2

离线下载的功能基本就搭建好了。但是，服务器上的东西怎么拉回本地呢？

两个办法，搭建FTP或者HTTP服务。前者安装vsftpd，后者安装apache2。因为后者结合Chrome浏览器可以有在线观看视频的功能，笔者选择安装apache2。

apt-get install apache2

Apache默认的根目录路径为 /var/www/html，这不符合笔者的需要，打开 /etc/apache2/sites-available/000-default.conf，作出如下修改：

#ServerName 127.0.0.1
ServerAdmin webmaster@localhost
DocumentRoot /home/Downloads
<Directory "/home/Downloads">
Options Indexes FollowSymLinks
AllowOverride None
Require all granted
</Directory>

私人的离线下载服务器到这里算是正式完工。

下周有时间的话，把 Google BBR 和 Resilio Sync也搞了。
