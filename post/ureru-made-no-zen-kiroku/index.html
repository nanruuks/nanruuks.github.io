趁忘前写下的Hugo折腾全记录

July 2, 2018

上一篇文里提到，笔者在某个深夜里扣动鼠标开始了Blog的折腾之旅。“既然汪先生用的是JustWriting，那我也照着依样画葫芦做一个不就好了”心里这样想着，手里的鼠标却拖不动了。无他，域名太贵备案太麻烦罢了。有没有什么背靠大树好乘凉的方案呢？搜索引擎的众多答案最终都指向了 Github+Hugo 的方案。

就你了。

首先安装Linux。

在Windows 10下打开开发者模式并添加 Windows Subsystem for linux (Beta) 的功能，重启后进入 Microsoft store 安装Debian (被Centos劝退又不够内存上Ubuntu，笑)。安装过程很快，几分钟就完事了。赶紧运行试试看！打开开始菜单，直接敲键盘搜索bash，回车，然后按照提示配置好帐号和密码，Linux的部分就算完成了。要注意的问题是，Linux子系统的根目录在是在Windows 10的这个位置：

C:\Users\username\AppData\Local\Packages\TheDebianProject.DebianGNULinux_76v4gfsz19hv4\LocalState\rootfs\

紧接着安装Git：

cd /

sudo apt-get install git

配置好仓库后，就可以安装Hugo了：

sudo wget https://github.com/gohugoio/hugo/releases/download/v0.42.2/hugo_0.42.2_Linux-64bit.deb

sudo dpkg -i hugo_0.42.2_Linux-64bit.deb

我知道肯定会有小朋友问，为什么不直接 sudo apt-get install hugo 一下呢？答曰，版本太老，许多主题都用不上了。

然后新建一个站点：

hugo new site myname.github.io

接下来是一件极其重要的事情：选择主题！在这里挑一个自己喜欢的主题，然后下载并解压到 /themes 文件夹下。笔者选择的是tranquilpeak。

可定制的侧边栏很重要！可定制的侧边栏很重要！可定制的侧边栏很重要！重要的事情说三遍

安装完主题后，Blog的搭建才进行到一半，接下来需要修改配置文件 config.toml

稍稍偷个懒，直接复制 themes/hugo-tranquilpeak-theme/exampleSite 下的 cofig.toml 到站点根目录覆盖原有的config，逐条修改参数。下面是笔者圈出的比较重要的参数修改：

hasCJKLanguage = true 

clearReading = true 

hierarchicalCategories = false 

sidebarBehavior = 5 

swapPaginator = true

配置完后，Blog的主体架构就差不多完成了，赶紧测试一下：

hugo new post/testing.md

切回到 content/post 文件夹，只见 testing.md 就静静地躺在那里。

hugo server

一个静态Blog就这么初步建成啦(๑•̀ㅂ•́)و✧

浏览器打开 127.0.0.1:1313 发现没有什么疏漏的地方，渲染发布：

hugo -t hugo-tranquilpeak-theme

cd public

git init

git add . 

git commit -m "Initial commit." 

git remote add origin git@github.com:myname/myname.github.io.git

git push -u origin master
